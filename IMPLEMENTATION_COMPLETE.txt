HOW IT WORKS
â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•â•

1. AGENT EXECUTION
   Agent generates output using AI

2. KNOWLEDGE STORAGE
   Agent stores output in Qdrant vector database with metadata

3. USER FEEDBACK
   User accepts/rejects/modifies output and rates it

4. FEEDBACK LEARNING
   Feedback is stored in PostgreSQL + Qdrant
   Agent learns from the feedback automatically

5. CONTEXT RETRIEVAL
   Next time agent is called, it:
   - Retrieves similar past cases from vector DB
   - Uses them as context to improve generation
   - Result is better informed by learned patterns

6. PERFORMANCE TRACKING
   Background tasks:
   - Update agent performance metrics
   - Calculate acceptance rates
   - Detect trends (improving/declining/stable)
   - Generate recommendations

7. ANALYTICS
   Dashboard can show:
   - Overall AI evolution status
   - Agent-specific performance
   - Feedback patterns
   - Learning trends
